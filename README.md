# RecommendationPasswordSystem-GUI - Рекомендательная система проверки стойкости паролей

Программный инструмент с графическим интерфейсом для оценки практической стойкости паролей. В отличие от формальных проверок (длина, наличие спецсимволов), система оценивает устойчивость пароля к реальным сценариям атак: словарным, с правилами подстановки, по клавиатурным шаблонам и комбинаторным методам.

## Возможности

- **Многофакторный анализ** — 5 независимых модулей оценки, каждый ориентирован на конкретный тип атаки
- **Обратная связь** — подробное объяснение обнаруженных уязвимостей и конкретные рекомендации по улучшению пароля
- **Поддержка Hashcat** — загрузка словарей и файлов правил (.rule) в формате Hashcat
- **Графический интерфейс** — современный интерфейс с индикатором стойкости и детальной разбивкой по модулям
- **Анализ в реальном времени** — оценка обновляется по мере ввода пароля (с возможностью ручной проверки по кнопке)

## Модули анализа

### 1. Базовый анализ строки
Оценивает длину пароля, состав символов (строчные, заглавные, цифры, спецсимволы), разнообразие уникальных символов, наличие повторов и монотонных паттернов чередования.

### 2. Словарный анализ
Проверяет пароль на полное совпадение, совпадение без крайних цифр/символов и наличие словарных слов как подстрок. Использует встроенный словарь (~800 популярных паролей) или загружаемый пользователем файл словаря.

### 3. Анализ правил (Rule-Based)
Выявляет типичные преобразования: leet-speak замены (`a→@`, `o→0`, `s→$`), капитализацию первой буквы, добавление цифр, спецсимволов и годов. Выполняет обратные преобразования и проверяет базовое слово по словарю. Таким образом, `P@ssw0rd!` распознаётся как модификация `password`. Поддерживает загрузку файлов правил Hashcat (`.rule`) для расширенного анализа — парсер умеет обрабатывать и обращать операции `:`, `l`, `u`, `c`, `t`, `r`, `d`, `f`, `q`, `$`, `^`, `s`, `z`, `Z`, `p` и другие.

### 4. Клавиатурные шаблоны
Обнаруживает горизонтальные (`qwerty`, `asdfgh`), вертикальные (`1qaz`, `2wsx`) и диагональные клавиатурные прогулки, включая Shift-варианты (`!QAZ@WSX`) и шаблоны NumPad. Использует пространственный анализ смежности клавиш на раскладке QWERTY.

### 5. Структурный и комбинаторный анализ
Выявляет структурные паттерны: «слово + число» (`admin123`), «имя + год» (`john2024`), конкатенацию двух словарных слов (`hellokitty`), повторяющиеся блоки (`abcabc`).

## Система оценки

Итоговая оценка формируется на основе взвешенной суммы результатов всех модулей:

| Модуль | Вес |
|--------|-----|
| Базовый анализ строки | 15% |
| Словарный анализ | 30% |
| Анализ правил | 20% |
| Клавиатурные шаблоны | 15% |
| Структурный и комбинаторный анализ | 20% |

| Оценка | Уровень стойкости |
|--------|-------------------|
| 0–19 | Очень слабый |
| 20–39 | Слабый |
| 40–59 | Средний |
| 60–79 | Надёжный |
| 80–100 | Очень надёжный |

При обнаружении критической уязвимости (прямое совпадение со словарём) итоговая оценка ограничивается сверху, не позволяя компенсировать слабость одного модуля высокими баллами других.

## Установка

### Требования

- Python 3.8+
- ОС Windows / macOS / Linux

### Установка зависимостей

```bash
pip install -r requirements.txt
```

Единственная внешняя зависимость — `customtkinter`.

## Запуск

```bash
python main.py
```

## Использование

1. **Ввод пароля** — введите пароль в поле ввода. По умолчанию анализ выполняется автоматически по мере набора (чекбокс «Автоматическая проверка» включён). Для ручного режима снимите чекбокс и нажимайте кнопку «Проверить».
2. **Просмотр результатов** — индикатор стойкости показывает общую оценку, в левой панели — детальная разбивка по модулям.
3. **Рекомендации** — в правой панели отображаются приоритизированные рекомендации по улучшению пароля.
4. **Загрузка словаря** — нажмите «Загрузить словарь» для загрузки файла словаря в формате Hashcat (текстовый файл, одно слово на строку).
5. **Загрузка правил** — нажмите «Загрузить правила» для загрузки файла правил Hashcat (`.rule`).

## Формат файлов

### Словарь (`.txt`)

Текстовый файл, одно слово/пароль на строку:

```
password
123456
qwerty
admin
letmein
```

### Правила Hashcat (`.rule`)

Файл правил Hashcat. Каждая строка — одно правило (последовательность операций преобразования). Строки, начинающиеся с `#`, игнорируются.

Поддерживаемые операции:

| Операция | Описание | Пример |
|----------|----------|--------|
| `:` | Без изменений | `hello` → `hello` |
| `l` | Все строчные | `Hello` → `hello` |
| `u` | Все заглавные | `hello` → `HELLO` |
| `c` | Капитализация | `hello` → `Hello` |
| `r` | Реверс | `hello` → `olleh` |
| `d` | Дублирование | `hello` → `hellohello` |
| `$X` | Добавить символ X в конец | `hello` + `$1` → `hello1` |
| `^X` | Добавить символ X в начало | `hello` + `^!` → `!hello` |
| `sXY` | Заменить X на Y | `hello` + `se3` → `h3llo` |
| `zN` | Дублировать первый символ N раз | `hello` + `z2` → `hhhello` |
| `ZN` | Дублировать последний символ N раз | `hello` + `Z2` → `hellooo` |

Операции можно комбинировать: `c$1` = капитализация + добавить «1» (`hello` → `Hello1`).

Пример файла:

```
:
l
u
c
d
$0
$1
$0$0
c$1
```

## Примеры работы

| Пароль | Оценка | Уровень | Причина |
|--------|--------|---------|---------|
| `password` | 20 | Слабый | Прямое совпадение со словарём |
| `P@ssw0rd!` | 35 | Слабый | Leet-speak модификация словарного слова |
| `qwerty123` | 20 | Слабый | Клавиатурный шаблон + словарь |
| `admin2026` | 35 | Слабый | Словарное слово + год |
| `Xk9#mL2$vQp!` | 95 | Очень надёжный | Случайная комбинация символов |
